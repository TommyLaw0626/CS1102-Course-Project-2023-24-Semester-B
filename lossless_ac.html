Arithmetic Coding

  • While Huffman code uses discrete numbers to represent the code, arithmetic coding uses continuous numbers that assign fractional values to the symbols based on their probabilities.
  • Arithmetic coding needs more computational resources, but it generates higher compression ratios compared to Huffman.
  • Use for JPEG 2000.

    • Example:



    Difference=upper limit - lower limit
    Range=lower limit to lower limit + difference (probability of symbol)
    Output: [0.6607,0.66303]

    As shown by the example, Arithmetic coding has a higher error rate compared to Huffman coding due to the detailed decimals. If the decimal is wrong, it will generate a different final output.
