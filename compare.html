Comparison between Lossy and Lossless Image Compression

Function:
  • Lossless: restores, rebuilds compressed file data in original form
  • Lossy: file data removed permanently to reduce the data size
What type of files to apply:
  • Lossless: files where data loss is unacceptable, where information loss can be significant and cause an issue
  • Lossy: files where data loss is acceptable
Format examples:
  • Lossless: GIF, RAW, BMP, PNG
  • Lossy: JPEG
Algorithms used:
  • Lossless: Run Length Encoding, Lempel-Zic-Welch, Huffman Coding, Arithmetic Encoding
  • Lossy: Transform Coding, Discrete Cosine Transform, Discrete Wavelet Transform, Fractal Compression
Advantages:
  • Lossless: Keeps file quality in smaller size
  • Lossy: File size reduces significantly, Compression supported by various tools, plugins and software, and can choose preferred strength of compression
Disadvantages:
  • Lossless: The file size after compression is comparatively large
  • Lossy: Degradation, Original file quality cannot be recovered with decompression
